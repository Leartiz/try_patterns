# Greedy

**Жадный алгоритм** на каждом шаге выбирает локально лучший вариант, не откатываясь назад.

Работает не всегда. Нужно понимать (или доказать), что локальный выбор не портит глобальный ответ.

## Когда подходит

1. **Greedy choice** - можно взять лучший шаг сейчас и не потерять оптимум.
2. **Optimal substructure** - после выбора остается та же задача меньшего размера.

Типичный шаблон: сортировка + один проход, на каждом шаге - лучший допустимый выбор.

## Когда ломается

Если "съесть" сейчас что-то крупное мешает лучшему решению потом.

Пример: монеты `[1, 3, 4]`, сумма `6`.
- greedy (всегда наибольшая монета): `4 + 1 + 1` -> 3 монеты
- оптимум: `3 + 3` -> 2 монеты

Тогда нужны DP или другой подход.

## Greedy vs DP

| | Greedy | DP |
|---|--------|-----|
| Выбор на шаге | один | перебор, запоминание |
| Сложность | часто `O(n log n)` | часто больше |
| Когда | есть доказательство жадного выбора | подзадачи перекрываются, greedy не доказуем |
